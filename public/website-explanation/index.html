Glitch Portfolio Website Explanation

A web version of the handoff document covering the portfolio URL, hosting model, project location, React/Vite structure, deployment workflow, and verification checks.

Bottom line: the public GitHub Pages link does not turn off when this PC turns off. GitHub serves that public site. The local localhost link only works while the Vite dev server is running, because computers still insist on being powered on to do local computer things.

Public Link vs. Local Link

Link Purpose Turns off with PC?
https://ponasytfun.github.io/portfolio/ Public production site hosted by GitHub Pages. No. GitHub serves it.
http://localhost:5174/portfolio/ Local development copy served by Vite on this PC. Yes. It needs the local server.
http://127.0.0.1:5174/portfolio/ Same local development copy, using the loopback IP instead of localhost. Yes. Same local server.

Deployment

GitHub Actions builds the Vite app and deploys the generated dist folder to GitHub Pages.

Technology Stack

React

The portfolio UI is assembled from reusable JSX components.

Vite

Vite provides the local dev server, production build, and static output for GitHub Pages.

Global CSS

src/styles.css and src/redesign.css define the dark pink/purple visual system.

Static Hosting

No backend server is required for the public site. GitHub Pages serves static HTML, CSS, JS, and assets.

Source Structure

Path Role
src/App.jsx Composes the visible page from Nav, Hero, About, StaffRoles, SpecializedSkills, Contact, and Footer.
src/data/portfolioData.js Central editable source for the visible portfolio copy, roles, skills, and Discord username.
vite.config.js Sets base: "/portfolio/" so GitHub Pages asset paths work under the repository URL.
.github/workflows/deploy.yml Builds with Node 24 and deploys the static app to GitHub Pages on pushes to main.
public/docs Stores the downloadable Word document version of this explanation.

Visible Site Sections

  • Fixed top navigation with profile avatar and section links.
  • Hero section with profile image, Glitch headline, Minecraft Developer title, and contact/about buttons.
  • About section with profile paragraph and summary stats.
  • Current and past staff role cards for SMP projects.
  • Specialized skills cards for plugins, server operations, Discord tools, automation, UI, debugging, hosting, and community work.
  • Contact section with a Discord username copy button.
  • Footer with links back to the top and to this explanation page.
